# 規則總表（Clawdbot / molt）

這頁是「我被要求遵守的明確規則」彙整，目的：**降低遺忘/誤解**，也方便你隨時稽核與更新。

> 註：這裡列的是「可操作的工作規則」，不是個人隱私推論；若你希望新增/刪除任何規則，直接在 Telegram 跟我說即可。

---

## A) 觸發規則（你丟訊息＝我應該做什麼）

### A1) YT-澔哥（游庭皓）直播摘要 → 寫入 stock_report 網站
- **觸發條件**：你在 Telegram 丟出以 **「今日直播摘要」** 開頭的內容。
- **我必須做的事**：
  1) 把該段摘要寫入 `stock_report` 的 **YT-澔哥週檔**（`reports/youtube/YYYY-Www.md` 與 `docs/reports/youtube/YYYY-Www.md`）
  2) **commit + push**，以觸發 GitHub Pages 更新
- **段落標題格式**：`YYYY-MM-DD (Weekday)`（例：`2026-03-04 (Wed)`）

## B) Heartbeat / Cron 規則

### B1) Moltbook API key：拿到之前不要再問
- **規則**：你說會在 **晚上 9 點以後（Asia/Taipei）**再提供 Moltbook API key；在拿到 key 前，我在 heartbeat **不得再檢查/詢問 key**。
- **拿到 key 後**：
  1) 讀取 <https://www.moltbook.com/heartbeat.md> 並依指示檢查
  2) 更新 `memory/heartbeat-state.json` 的 `lastMoltbookCheck`（epoch seconds）

## C) 內容與格式規則（投資/報告）

### C1) 語言與風格
- 預設使用 **繁體中文**。
- 盡量省 token；避免不必要的閒聊/重複敘述。
- 語氣偏研究（偏研究導向）。

### C2) 股票代碼出現時要補公司名
- 只要在敘事中提到代號/代碼，必須用 **公司名(代碼)** 或 **代碼 公司名**（例如：台積電(2330)）。

### C3) 每日摘要應包含的元素（偏好）
- 價格、成交量、關鍵新聞
- 族群/輪動（3–5 點）
- 台股加掛：外資/投信/自營商買賣超變化
- 題材候選：分「穩健 / 進攻」，進攻需過流動性濾網（20 日均量 < 5,000 張排除）

## D) 寫入/推送（git）工作流規則

### D1) 任何「寫入+push」要有可追溯變更
- 文件要能對應到變更原因（必要時更新 changelog）。
- 若涉及較大改動：優先遵循既有流程（常見做法：pull → 寫 docs → changelog → 再 pull/rebase → commit → push）。

---

## E) CC研究工具（個股分析）

### E1) 觸發：你指定「請你用 CC 研究工具分析」+ 股票名稱(代碼)
- **觸發條件**：你明確說「請你用 CC 研究工具分析」並點名個股（格式：`股票名稱(數字股票代碼)`，例如：`台積電(2330)`）。
- **我必須做的事**：用 CC 研究工具輸出該個股報告。
- **預設輸出風格**：相對更詳盡（約 **500 字**）且可保留表格；避免不必要的閒聊。
